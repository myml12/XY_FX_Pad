# LED Visual シミュレーター

アプリの `LED Visual` は、FX Padの四辺に取り付ける112個のアドレサブルLEDを画面上で再現する機能です。音を鳴らさずにパッドをマウスで操作しても、実機の演出を確認できます。

## 操作と光の対応

| 入力 | LED Visualでの見え方 |
| --- | --- |
| X / Y位置 | 4点荷重から求めた重心に近いLEDほど明るくする。角に寄ると、隣り合う二辺も自然に光る。 |
| 筆圧 | Pressure Meterと同じ「すぐ上がり、ゆっくり下がる」感覚で、明るさと光の広がる半径を増やす |
| 移動 | 前の位置の残光が消える前に次の光が加わり、流れるように見える |
| 筆を離す | 残光だけを滑らかに減衰させる |

`LED Visual (Blue Pressure)` でオン／オフを切り替えます。現在は青1色で、明るさだけが筆圧に追従します。色相の変化やBPM連動は、基本の追従感を確認してから追加できるよう、描画と演出計算を分離しています。

## 実機への移植

画面の1ピクセルは、実機のLED 1個に対応します。ESP32側では80HzのHX711読取りと別に、30〜40Hzでこの演出を更新します。現在の演出が各フレームで使う値は `x`, `y`, `pressure`, `touching` の4つだけです。

各LEDと筆の重心を同じXY座標系に置き、その直線距離に応じたガウス状の明るさを足します。各フレームで残光を減衰させるため、実機ではAPA102 / SK9822を112個程度の1本の連結として配置すれば、この見え方をほぼそのまま再現できます。

> 座標の向き: ControllerBridgeのYは上が `1`、画面描画のYは上が `0` です。LED Visualではこの差を反転してから距離計算するため、`raw0=右上 → raw1=左上 → raw2=左下 → raw3=右下`（反時計回り）の物理配置と一致します。
